# Стиль — PWA (P0)

Личный цифровой гардероб + (позже) ИИ-стилист для Екатерины. Веб-PWA, ставится на iPhone
через «Добавить на экран». Один пользователь, работает офлайн, без App Store.

**Живёт на https://sizur.xyz** (GitHub Pages через Actions, домен на Spaceship).
Вход за PIN-занавесом — код в `src/lib/gate.js` (`ACCESS_PIN`). Это занавес от случайных,
не настоящий замок: статические `/pictures` и `wardrobe_export.json` отдаются напрямую.
`robots.txt` + `noindex` держат сайт вне поисковиков.

Построено на реальных мигрированных данных GetWardrobe (`../wardrobe-migration/`):
**291 вещь · 157 образов с раскладкой канваса · 456 вырезок WebP (~32 МБ)**.

## Стек
- **Svelte 5** (руны) + **Vite** — маленький бандл, быстрый холодный старт на телефоне.
- **Konva** — канвас-редактор образов (drag / scale / rotate / слои / зеркало).
- **Dexie** (IndexedDB) — локальное хранилище правок; картинки лежат статикой и кэшируются SW.
- **vite-plugin-pwa** (Workbox) — манифест + service worker, прекэш гардероба для офлайна.

## Что работает (P0)
- **Гардероб** — сетка вещей, фильтры категория / сезон / цвет (палитра кластеризуется из данных), счётчик «N из N».
- **Карточка вещи + редактирование** — фото, название, сезоны (метка поверх, не прячет), бренд, «на выброс/архив», «в каких образах» с переходом в образ; правки персистятся.
- **Образы** — галерея 157 коллажей.
- **Канвас-редактор** — открывает реальный образ 1:1, перемещение/масштаб/поворот/слои/зеркало/дубликат/удаление, добавление вещи из каталога, **undo/redo**, сохранение в IndexedDB с перерендером превью (WebP).
- **Чемоданы** — список (дом/спорт, кишинёв 10.25 с вещами+образами, «на выброс» приглушён), детальный экран с сегментом Вещи/Образы; «на выброс» остаётся в поиске.
- **Сегодня** — герой-образ по текущему сезону, честное текстовое обоснование (без фейк-score), Надеть / Другой вариант / Собрать самой.
- **Стилист — Style DNA** — палитра с %, силуэты/типы, тир бренда — всё вычислено из её данных. ИИ-генерация честно помечена «скоро · P1».
- **Календарь** — назначение образа на дату (отдельный стор Dexie), превью образа в дне.
- **Вещь с фото** — съёмка/выбор фото → **удаление фона в браузере** (`@imgly/background-removal`, WASM, без сервера) → вырезка → новая вещь в гардеробе.
- **PWA-обёртка** — устанавливается на домашний экран, офлайн (прекэш ~32 МБ).

## Запуск
```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # прод-сборка в dist/ (+ service worker, прекэш ~32 МБ)
npm run preview  # предпросмотр прод-сборки
```
Данные сидируются в IndexedDB при первом запуске из `public/wardrobe_export.json`.
Сброс/пересев — поднять `SEED_VERSION` в `src/lib/db.js`.

## Структура
```
public/
  wardrobe_export.json     # миграция (291 вещь / 157 образов / 4 чемодана)
  pictures/<guid>.webp     # 456 вырезок, привязка 100%
  icons/                   # PWA-иконки
src/
  lib/
    catalog.js             # хелперы домена (картинки, сезоны, кластеризация палитры)
    db.js                  # Dexie + сидирование + saveOutfit
    state.svelte.js        # глобальный UI-стейт (руны)
  components/
    Wardrobe.svelte        # каталог + фильтры
    Outfits.svelte         # галерея образов
    CanvasEditor.svelte    # Konva-редактор (ядро паритета)
    ItemPicker.svelte      # шит «добавить вещь»
    ItemCard.svelte / TabBar.svelte / Stub.svelte
```
